# Gradio 版本兼容性修复说明

## 问题背景

由于 Gradio 库更新到 6.0+ 版本，部分 API 发生了变化，导致原有代码无法正常运行。

## 已修复的问题

### 1. ❌ `gr.Box` 组件已移除

**错误信息**:
```
AttributeError: module 'gradio' has no attribute 'Box'
```

**修复位置**: `ui/tabs/dataset_tab.py:70`

**修复方案**:
```python
# 修复前
recommendation_box = gr.Box(visible=False)

# 修复后
with gr.Group(visible=False) as recommendation_box:
    # ... 内容组件 ...
```

**说明**: Gradio 6.0+ 移除了 `Box` 组件，改用 `Group` 替代。

---

### 2. ❌ `show_copy_button` 参数已移除

**错误信息**:
```
TypeError: Chatbot.__init__() got an unexpected keyword argument 'show_copy_button'
```

**修复位置**: `ui/tabs/chat_tab.py:27`

**修复方案**:
```python
# 修复前
chatbot = gr.Chatbot(
    label="对话历史",
    height=500,
    show_copy_button=True,  # ❌ 此参数已移除
    avatar_images=(...)
)

# 修复后
chatbot = gr.Chatbot(
    label="对话历史",
    height=500,
    avatar_images=(...)
)
```

**说明**: 新版本中复制按钮已成为默认功能，无需手动启用。

---

### 3. ❌ `theme` 和 `css` 参数位置变更

**警告信息**:
```
UserWarning: The parameters have been moved from the Blocks constructor to the launch() method in Gradio 6.0: theme, css. Please pass these parameters to launch() instead.
```

**修复位置**: `ui/app.py`

**修复方案**:

**Blocks 定义部分**:
```python
# 修复前
with gr.Blocks(
    title="BioTransLM 可视化系统",
    theme=theme,  # ❌ 已移除
    css="""...""",  # ❌ 已移除
    analytics_enabled=False
) as app:

# 修复后
with gr.Blocks(
    title="BioTransLM 可视化系统",
    analytics_enabled=False
) as app:
```

**launch() 调用部分**:
```python
# 修复前
app.launch(
    server_name=server_name,
    server_port=server_port,
    **kwargs
)

# 修复后
custom_css = """
    .gradio-container {
        max-width: 1400px !important;
    }
    .main-header {
        text-align: center;
        margin-bottom: 20px;
    }
"""

theme = gr.themes.Soft(...)

app.launch(
    server_name=server_name,
    server_port=server_port,
    theme=theme,  # ✅ 添加到这里
    css=custom_css,  # ✅ 添加到这里
    **kwargs
)
```

### 4. ❌ `outputs` 列表中不支持 `None`

**错误信息**:
```
AttributeError: 'NoneType' object has no attribute '_id'
```

**修复位置**: `ui/tabs/inspection_tab.py:294`

**修复方案**:
```python
# 修复前
outputs=[
    gating_stats, gating_plot,
    multigran_info, None,  # ❌ None 不被支持
    ...
]

# 修复后
outputs=[
    gating_stats, gating_plot,
    multigran_info, multigran_info,  # ✅ 使用相同组件替代
    ...
]
```

**说明**: Gradio 6.0+ 要求所有输出组件必须是有效的 Block 对象。

---

### 5. ❌ `gr.load()` API 变更

**错误信息**:
```
TypeError: load() missing 1 required positional argument: 'name'
```

**修复位置**: 
- `ui/tabs/model_tab.py:494-495`
- `ui/tabs/inspection_tab.py:306`

**修复方案**:
```python
# 修复前
gr.load(fn=refresh_model_status, outputs=[model_status_box, model_info_json])

# 修复后
gr.on(
    triggers=[],
    fn=refresh_model_status,
    outputs=[model_status_box, model_info_json]
)
```

**说明**: `gr.load()` 被废弃，改用 `gr.on()` 配合空的 `triggers` 列表实现页面加载时触发。
