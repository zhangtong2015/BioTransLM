### 2️⃣ 🧠 模型管理

**用途**：创建、训练、保存和加载 AI 模型

#### 创建新模型

1. 点击 **"🧠 模型管理"** 标签页
2. 点击 **"➕ 创建新模型"** 按钮
3. 选择模型类型：
   - **BioGenerator** - 基础语言生成模型（推荐新手）
   - **Orchestrator** - 完整编排模型（高级用户）
4. 配置参数（可使用默认值）：
   - SDR 维度：4096（推荐）
   - 列数：4096（推荐）
   - 稀疏度：2%（推荐）
5. 点击 **"创建模型"**

#### 训练模型

**一键训练（推荐）**：
1. 先在 "📊 数据集管理" 上传数据
2. 系统会自动推荐训练配置
3. 点击 **"🚀 使用推荐配置并开始训练"**

**手动训练**：
1. 在模型管理页面找到已创建的模型
2. 点击 **"开始训练"**
3. 设置训练参数：
   - Batch Size: 16（小数据集）~ 64（大数据集）
   - Learning Rate: 0.0001（推荐）
   - Epochs: 5-20（根据数据量调整）
4. 点击 **"开始训练"**

#### 保存/加载模型

**保存模型**：
1. 训练完成后，点击 **"💾 保存模型"**
2. 选择保存位置
3. 输入模型名称
4. 点击确认

**加载模型**：
1. 点击 **"📂 加载模型"**
2. 选择之前保存的模型文件
3. 点击确认
4. 状态显示 **"🟢 已加载"** 即可使用

#### 损失曲线图

实时显示 4 种损失的变化：
- **总损失**（蓝色）- 整体训练效果
- **LM 损失**（绿色）- 语言建模能力
- **稀疏度损失**（橙色）- SDR 稀疏性
- **时序损失**（红色）- 时序预测能力

**理想趋势**：所有曲线都应该逐渐下降并趋于平稳

#### HTM 指标

- **激活率** - 活跃列的比例（理想值：1-5%）
- **Burst 率** - 突发激活的比例（理想值：< 10%）
- **预测准确率** - 时序预测准确度（越高越好）

#### 控制按钮

- **⏸️ 暂停训练** - 暂时停止，可恢复
- **▶️ 继续训练** - 从暂停处继续
- **⏹️ 停止训练** - 完全停止，不可恢复

## ❓ 常见问题

### Q1: 启动后浏览器显示空白页？

**A**: 等待 10-30 秒，程序需要初始化。如果仍然空白：
1. 检查命令行是否有错误信息
2. 刷新浏览器页面（F5）
3. 关闭程序重新启动

### Q2: 对话时提示"未加载模型"？

**A**: 需要先创建和加载模型：
1. 点击"🧠 模型管理"
2. 创建新模型（如果没有）
3. 点击"加载模型"选择已保存的模型
4. 看到"🟢 已加载"状态后再去对话

### Q3: 训练时提示"CUDA out of memory"？

**A**: 显存不足，解决方法：
1. 减小 Batch Size（改为 8 或 4）
2. 减小 Max Sequence Length（改为 256）
3. 使用 CPU 模式（速度较慢但稳定）

### Q4: 上传数据后没有显示推荐配置？

**A**: 检查数据格式：
1. 确保数据符合支持的格式（JSONL/CSV/TXT/JSON）
2. 检查数据是否包含必要的字段
3. 查看数据统计，确认样本数量 > 10

### Q5: 训练完成后对话效果不好？

**A**: 可能需要更多训练：
1. 增加 Epochs（改为 15-20）
2. 增加训练数据量
3. 调整学习率（尝试 0.00005 或 0.0002）
4. 检查数据质量（是否有噪声或错误）

### Q6: 如何保存对话记录？

**A**: 目前不支持直接导出，但可以：
1. 截图保存重要对话
2. 复制粘贴到文本文件
3. 查看日志文件（在 `logs/` 目录）

### Q7: 程序运行很慢怎么办？

**A**: 优化建议：
1. 使用 GPU 加速（如果有 NVIDIA 显卡）
2. 减小模型规模（SDR 维度改为 2048）
3. 关闭其他占用资源的程序
4. 重启程序释放内存

## 💡 技巧与建议

### 新手入门路线

**第 1 天**：
1. 启动程序
2. 在"对话"页面测试内置模型（如果有）
3. 熟悉各个标签页的功能

**第 2 天**：
1. 准备一些简单的问答数据（50-100 条）
2. 在"数据集管理"上传
3. 使用推荐配置训练第一个模型

**第 3 天**：
1. 测试训练好的模型
2. 在"处理过程查看"观察 AI 思考过程
3. 根据效果调整训练参数

### 数据准备技巧

**高质量数据特征**：
- ✅ 格式统一、规范
- ✅ 内容准确、无错别字
- ✅ 覆盖常见场景
- ✅ 难度适中

**避免的问题**：
- ❌ 数据量太少（< 50 条）
- ❌ 格式混乱、不统一
- ❌ 包含大量噪声或错误
- ❌ 文本过长（> 1000 字）

### 训练参数调优

**小数据集**（< 500 条）：
```
Batch Size: 8-16
Learning Rate: 0.0001
Epochs: 15-20
```

**中等数据集**（500-5000 条）：
```
Batch Size: 16-32
Learning Rate: 0.0001
Epochs: 10-15
```

**大数据集**（> 5000 条）：
```
Batch Size: 32-64
Learning Rate: 0.00005
Epochs: 5-10
```

### 对话技巧

**获得更好回复的方法**：
1. 问题要具体明确
   - ✅ "什么是 HTM 理论？"
   - ❌ "给我讲讲那个理论"

2. 提供上下文
   - ✅ "我在学 HTM 理论，请问它的核心概念是什么？"
   - ❌ "核心概念是什么？"

3. 分步骤提问
   - 先问基础概念
   - 再问具体应用
   - 最后问深入问题

### 性能优化建议

**有 GPU（NVIDIA 显卡）**：
- 使用 CUDA 加速
- 可以使用更大的模型
- 训练速度快 10-50 倍

**只有 CPU**：
- 使用较小的模型（SDR 2048）
- 减小 Batch Size
- 训练时间较长，但结果一样好

**内存不足**：
- 关闭其他程序
- 减小序列长度
- 使用更小的模型
